Problem

Make network availability visible without manual checks.

The system keeps a managed host list, periodically collects response data and exposes operational statistics through a simple web interface. Docker Compose makes the frontend and backend reproducible as one unit.

01 / CRUD

Host management

Add, edit and remove validated IP addresses through a REST API.

02 / MONITORING

Background collection

An application worker performs repeated pings and stores latency, delivery and loss percentages.

03 / DATA FLOW

CSV workflows

Bulk host import and statistics export keep the tool useful outside its own interface.

Architecture

A small system with visible boundaries.

The interface consumes a documented REST surface. Persistence and monitoring stay behind the backend boundary, while Docker Compose defines the delivery topology.

Selected code

Validate the boundary before persistence.

The real endpoint uses Pydantic's IPvAnyAddress, rejects duplicates and returns the persisted SQLAlchemy entity through the response model.

backend/main.pyFastAPI
@app.post("/hosts", response_model=Host)
def add_host(host: Host, db: Session = Depends(get_db)):
    duplicate = db.query(HostDB).filter(
        HostDB.ip == str(host.ip)
    ).first()
    if duplicate:
        raise HTTPException(
            status_code=400,
            detail="IP already exists",
        )

    db_host = HostDB(id=host.id, ip=str(host.ip))
    db.add(db_host)
    db.commit()
    db.refresh(db_host)
    return db_host
